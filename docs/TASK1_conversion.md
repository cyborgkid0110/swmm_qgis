# TASK1_conversion.md — Data Standardization & SWMM Conversion Module

## Purpose

Build a conversion pipeline that transforms digitized drainage network datasets (CSV, XLSX, Shapefile) from multiple city databases into EPA SWMM hydraulic model input files (`.inp`).

---

## Main Tasks

1. **Standardize digitized datasets** — Unify inconsistent schemas, encodings, and coordinate systems from raw city databases into a standard ESRI Shapefile format.
2. **Convert standardized datasets to SWMM input** — Generate `.inp` files directly consumable by EPA SWMM.

---

## Dataset Conversion Pipeline

```
Raw datasets → migrate_all.py → Standardized CSV → standardize.py → Shapefile
                                                  → conversion.py  → SWMM .inp
```

  - Sewers: decompose LineStrings into junction+conduit segments with `coord_registry` dedup
  - Manholes: pre-seed `coord_registry` so sewer junctions inherit manhole names/attributes
  - Orifices: `Source` field specifies upstream sewer/canal route → `FromNode` merges with nearest junction on that route (excluding CD-prefix weir junctions); `Receiver` specifies downstream lake/canal/river → `ToNode` snaps to lake junction or nearest canal junction (excluding CD-prefix), offset by `LINK_OFFSET` if coincident with orifice position
  - Pumps: link from Source lake position → nearest sewer junction
  - Outfalls: replace canal endpoint junction via `SewerLine` (adopt junction name for consistency)
  - Subcatchments: find outlet on `SewerRoute` canal/sewer (fallback to nearest any junction)
  - Canal/river: route-tagged junctions, 5m endpoint snap for branch merging
  - Weirs: vertex weir `CD_dn` registered at original vertex coords for correct branch merging
