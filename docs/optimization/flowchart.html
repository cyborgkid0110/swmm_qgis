qEHVI Sedimentation Maintenance Optimization — Snake Flow Diagram

Flow 1

Flood Warning & Sedimentation Solution Proposal (qEHVI Optimization Loop)

Bayesian optimization loop using BoTorch qEHVI: scenario input → initial sampling → SWMM evaluation → GP surrogate fitting → qEHVI acquisition (BoTorch) → discretize → Pareto front update → convergence check. Decision variable x ∈ {0,1}ᴺ (binary maintenance selection for N sediment monitoring points), relaxed to [0,1]ᴺ for GP fitting.

Flow 2

Algorithm Effectiveness Evaluation (Pre vs. Post Maintenance)

Parallel dual-scenario execution comparing the original (pre-maintenance) network with the post-sedimentation-resolved network. Both run through SWMM independently and their KPI vectors are compared to quantify the optimization's effectiveness.

Objective Function — KPI Definitions

F₁ Flood Severity Index

Quantifies flooding at nodes weighted by importance:
F₁ = Σᵢ wᵢ · [α · Vᵢᶠˡᵒᵒᵈ/Vᵢʳᵉᶠ + β · Tᵢᶠˡᵒᵒᵈ/Tʳᵉᶠ]

SWMM outputs: flood volume Vᵢ (m³), flood duration Tᵢ (hrs) per node.
Weights: α + β = 1; wᵢ from land use / population density.

F₂ Drainage Capacity Index

Penalizes surcharge and low discharge across pipes:
F₂ = Σⱼ Lⱼ · [ζ·Iⱼ/Iⱼᶠᵘˡˡ + γ·Tⱼˢᵘʳᶜʰ/Tʳᵉᶠ − δ·Qⱼ/Qⱼᶠᵘˡˡ]

SWMM outputs: inflow ratio, surcharge duration, discharge ratio per link.
Weights: Lⱼ = pipe length (spatial weight); ζ, γ, δ tunable.

F₃ Sedimentation–Maintenance Index

Long-term operational burden from sedimentation:
F₃ = Σⱼ [μ · SⱼLⱼ/Cⱼᶜᵃᵖ + ν · Cⱼᵐᵃⁱⁿᵗ/Cⱼᶜᵃᵖ]

Sources: Sⱼ from SWMM WQ module or field survey (Di et al. SI).
Weights: μ, ν tunable coefficients.
